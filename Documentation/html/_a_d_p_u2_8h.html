◆ ADPU2_ProcessAndDispatch()

sys_error_code_t ADPU2_ProcessAndDispatch ( ADPU2_t *  _this)

Process an input data when it is ready, and dispatch the processed data: first all listeners are notified about the new data, then the new data is sent to the next DPU of the chain for further processing.

This method is called automatically by the DPU if it has not notify callback registered, otherwise it is responsibility of the application to call this method start the processing of the input data.

Parameters
_this[IN] specifies a pointer to the object.
Returns
SYS_NO_ERROR_CODE if success, SYS_ADPU2_NO_READY_ITEM_ERROR_CODE if there are no input data ready to be processed, others error code otherwise.

◆ ADPU2_SetInDataBuffer()

sys_error_code_t ADPU2_SetInDataBuffer ( ADPU2_t *  _this,
uint8_t *  p_buffer,
uint32_t  buffer_size 
)

Set the memory buffer used by the DPU to manage the input data. It must be big enough to store the payload of one or more input data. To know the size in byte of the payload of one input data, it is possible to use the method ADPU2_GetInDataPayloadSize(). If the DPU has already an input buffer, then it is released and replaced with the new one.

If buffer_size is equal to zero then the DPU releases the buffer resources.

If the DPU has no input data buffer then its behavior, when it receive new data, is not defined.

Parameters
_this[IN] specifies a pointer to the object.
p_buffer[IN] specifies a buffer .
buffer_size[IN] specifies the size in byte of the buffer.
Returns
SYS_NO_ERROR_CODE if success, SYS_OUT_OF_MEMORY_ERROR_CODE otherwise.

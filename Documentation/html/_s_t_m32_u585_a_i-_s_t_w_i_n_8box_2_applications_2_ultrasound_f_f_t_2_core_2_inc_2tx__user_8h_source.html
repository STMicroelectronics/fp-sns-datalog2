59/* Define various build options for the ThreadX port. The application should either make changes
60 here by commenting or un-commenting the conditional compilation defined OR supply the defines
61 though the compiler's equivalent of the -D option.
62
63 For maximum speed, the following should be defined:
64
65 TX_MAX_PRIORITIES 32
66 TX_DISABLE_PREEMPTION_THRESHOLD
67 TX_DISABLE_REDUNDANT_CLEARING
68 TX_DISABLE_NOTIFY_CALLBACKS
69 TX_NOT_INTERRUPTABLE
70 TX_TIMER_PROCESS_IN_ISR
71 TX_REACTIVATE_INLINE
72 TX_DISABLE_STACK_FILLING
73 TX_INLINE_THREAD_RESUME_SUSPEND
74
75 For minimum size, the following should be defined:
76
77 TX_MAX_PRIORITIES 32
78 TX_DISABLE_PREEMPTION_THRESHOLD
79 TX_DISABLE_REDUNDANT_CLEARING
80 TX_DISABLE_NOTIFY_CALLBACKS
81 TX_NOT_INTERRUPTABLE
82 TX_TIMER_PROCESS_IN_ISR
83
84 Of course, many of these defines reduce functionality and/or change the behavior of the
85 system in ways that may not be worth the trade-off. For example, the TX_TIMER_PROCESS_IN_ISR
86 results in faster and smaller code, however, it increases the amount of processing in the ISR.
87 In addition, some services that are available in timers are not available from ISRs and will
88 therefore return an error if this option is used. This may or may not be desirable for a
89 given application. */
90

92 to tx_port.h for descriptions on each of these options. */

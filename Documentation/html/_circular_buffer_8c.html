CircularBuffer * CB_Alloc ( uint16_t  item_count)

Allocate an object of CircularBuffer type. This allocator implement the singleton design pattern, so there is only one instance of circular buffer. A new allocated object must be initialized before use it.

Parameters
[IN]nItemCount specifies the maximum number of items that is possible to store in the buffer.

Get a free item from the head of the buffer. A free item can be used by the caller to produce its content. When the item is ready the caller must call CBSetItemReady() to mark the item a ready to be consumed.

p_item[OUT] pointer to the new circular buffer item object. If the operation fails the pointer will be set to NULL.
Returns
SYS_NO_ERROR_CODE if success, SYS_CB_FULL_ERROR_CODE if the buffer is full.

uint16_t CB_GetReadyItemFromTail ( CircularBuffer *  _this,
CBItem **  p_item 
)

Get a ready item from the tail of the buffer. A ready item can be consumed by the caller. After the item is consumed the caller must call CBReleaseItem() in order to free the item.

Parameters
_this[IN] specifies a pointer to a CircularBuffer object.
p_item[OUT] pointer to the ready circular buffer item object. If the operation fails the pointer will be set to NULL.
Returns
SYS_NO_ERROR_CODE if success, SYS_CB_NO_READY_ITEM_ERROR_CODE if there are not ready item in the tail of the buffer.

◆ CB_ReleaseItem()

uint16_t CB_ReleaseItem ( CircularBuffer *  _this,
CBItem *  p_item 
)

Release an item. After an item has been consumed it must be released so it is marked as free, and avoid the buffer to become full. If the item is new, that means it has been allocate but it is not ready yet, the function fails.

Parameters
_this[IN] specifies a pointer to a CircularBuffer object.
p_item[IN] specifies a pointer to the item to be released.
Returns
SYS_NO_ERROR_CODE if success, SYS_CB_INVALID_ITEM_ERROR_CODE otherwise.

Generic/System error codes

The error codes as reported by the different Protocol layers. They start with 0x80

#define BLE_STATUS_UNKNOWN_CONNECTION_ID   ((tBleStatus)(0x80))
 The Connection Identifier does not exist. Temporary remapped to corresponding Controller Error.
 
#define BLE_STATUS_FAILED   ((tBleStatus)(0x81))
 The Host failed while performing the requested operation.
 
#define BLE_STATUS_INVALID_PARAMS   ((tBleStatus)(0x82))
 Invalid parameters passed at Host layer.
 
#define BLE_STATUS_BUSY   ((tBleStatus)(0x83))
 The Host is already processing another request received in advance.
 
#define BLE_STATUS_PENDING   ((tBleStatus)(0x84))
 The operation requested cannot be completed immediately by the Host (usually because of lack of resources). The operation is generally put on hold by the caller and it's usually retried on later time.
 
#define BLE_STATUS_NOT_ALLOWED   ((tBleStatus)(0x85))
 The requested operation cannot be performed by the Host in the current status.
 
#define BLE_STATUS_ERROR   ((tBleStatus)(0x86))
 The requested operation violates the logic of the called layer/function or the format of the data to be processed during the operation.
 
#define BLE_STATUS_OUT_OF_MEMORY   ((tBleStatus)(0x87))
 The requested operation failed because of lack of memory. Out of memory shall be returned for situations where memory will never become available again.
 
#define BLE_STATUS_INSUFFICIENT_RESOURCES   ((tBleStatus)(0x88))
 The requested operation failed for a temporary lack of resources (e.g. packet pool or timers), but it may be retried later when resources may become available (packets or timers may have been released by other consumers).
 
#define BLE_STATUS_NULL_PARAM   ((tBleStatus)(0x89))
 A NULL poiter was passed as function parameter.

Note
Error Codes are defined according to Bluetooth Specifications Vol.2 Part D.
  • HCI/Controller standard codes range is [0x00 : 0x3F]. Their name is prefixed by "BLE_ERROR_"
  • Host specific (and proprietary) error codes range is [0x80 : 0xFF]. Their name is prefixed by "BLE_STATUS_" and they're grouped by Layer
  • BLE_STATUS_SUCCESS value is 0x00 and is the unique error code used by both Controller and Host layers.
